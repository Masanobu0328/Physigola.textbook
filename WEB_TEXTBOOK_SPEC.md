# Physio Golazo Web教科書 制作仕様書

**最終更新日:** 2026-01-12
**重要度:** 最高（Strict Compliance Required）

本ドキュメントは、Physio Golazo Web教科書の制作における絶対的なルールを定義するものです。AIおよび開発者は、本仕様書の内容を遵守しなければなりません。

---

## 1. コンテンツの取り扱いに関する絶対ルール（最重要）

### 1.1 一言一句の完全維持
*   **原文の厳守:** 元となるMarkdownファイル（`revised/`フォルダ内のファイル）のテキストは、**一言一句変更してはならない**。
*   **禁止事項:**
    *   AIによる要約、意訳、表現の変更。
    *   「読みやすさ」を理由とした文末や接続詞の自己判断による修正。
    *   情報の省略、追加。
*   **例外:** ユーザーから明示的に「文章を修正して」と指示があった場合のみ対応可とする。

### 1.2 参考文献と引用の保全
*   **完全なる転記:** 参考文献リスト、引用元URL、PMIDなどは、元のMarkdownファイルから**完全に正確に**コピーしなければならない。
*   **順序と形式:** 番号順（例：【1】, 【2】...）や記載フォーマットを勝手に変更してはならない。これは学術的な信頼性を担保するための生命線である。

---

## 2. デザイン・UI/UX仕様

### 2.1 デザインコンセプト
*   **モバイルファースト:** スマートフォンでの閲覧体験を最優先する。
*   **プロフェッショナル:** 信頼感のある、清潔で洗練されたデザイン（「Physio Golazo」ブランドカラー：オレンジ #E09000 を基調）。
*   **脱・素人感:**
    *   **絵文字の完全禁止:** 本文、UI、ボタン等、いかなる場所にも絵文字（🐶, 📚, ✨等）を使用してはならない。アイコンが必要な場合はCSSシェイプまたはSVG画像を使用する。

### 2.2 デバイス対応要件
*   **スマートフォン:**
    *   片手での操作性（ボタンサイズ、タップ領域の確保）。
    *   横スクロールが発生しないレイアウト（テーブルは`table-responsive`で対応）。
*   **タブレット / PC:**
    *   コンテンツ幅を適切に制限（max-width: 900px）し、間延びしないレイアウト。
    *   十分な余白（ホワイトスペース）による高級感の演出。

### 2.3 印刷 / PDF化対応
*   **A4出力最適化:** ブラウザの印刷機能でA4 PDFとして出力した際、レイアウトが崩れないこと。
*   **不要要素の非表示:** 印刷時にはヘッダー、ナビゲーション、ボタン等のUI要素を自動的に非表示にする（`@media print`使用）。
*   **改ページ制御:** 章タイトル、図表、注釈ボックスの途中でページが切れないようCSSで制御する（`break-inside: avoid`）。

---

## 3. 技術仕様

### 3.1 ファイル構成
*   **HTML:** 各章ごとに独立したHTMLファイルを作成（例：`site/chapters/chapter-01.html`）。
*   **CSS:** 全ページ共通のスタイルシート（`site/css/style.css`）で管理し、デザインの一貫性を保つ。

### 3.2 HTMLマークアップ
*   **セマンティック:** 原文の構造（`#`→`h1`, `##`→`h2`等）を忠実にHTMLタグにマッピングする。
*   **クラス命名:** 汎用的で理解しやすいクラス名を使用する（例：`.note-box`, `.reference-item`）。

---

## 4. 作業プロセス

1.  **原文の読み込み:** 対象のMarkdownファイルを読み込む。
2.  **HTML変換:** テキストを変更せず、指定のHTMLテンプレートに流し込む。
3.  **検証:** 生成されたHTMLと元のMarkdownを比較し、テキスト（特に参考文献）に一切の差異がないことを確認する。
4.  **デザイン適用:** テキストには触れず、CSSクラスの適用のみを行う。
